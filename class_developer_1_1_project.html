Detailed Description

Container type for GP projects, allowing for monitoring and updating project files.

The general structure of projects is as follows:

  Project:
      - project file: (.gpp)
          - version information
          - rule/program/graph locations
          - run profiles
      - rules:
          - rule files (.gpr)
      - programs:
          - program files (.gpx)
      - graphs
          - graph files (.dot, .gxl)

An example of a simple project file follows, the format is a simple dialect of XML containing the necessary information to locate all of the project's files.

  <?xml version="1.0" encoding="UTF-8" ?>
  <project name="Project1" directory="/home/user/gpdeveloper/project1">
      <rules>
          <rule>rule1.gpr</rule>
          <rule>rule2.gpr</rule>
          <rule>rule3.gpr</rule>
      </rules>
      <programs>
          <program>program1.gpx</program>
      </programs>
      <graphs>
          <graph>graph1.gxl</graph>
          <graph>graph2.dot</graph>
      </graphs>
      <runconfigurations>
          <runconfiguration name="run1" program="program1" />
      </runconfigurations>
  </project>
